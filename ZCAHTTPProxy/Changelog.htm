1.2.1 бета

  • Достигнута полная совместимость прокси с проигрывателями WMP, MPC-HC, а также c медиацентрами WMC и MediaPortal.

1.2 пред бета

  • Стал поддерживать до 4-ёх одновременных сетевых tcp подключений.
  • Разработана новая уникальная технология кэширования потока тв канала с ограничением в 64 мегабайта, кодовое название MemoryCache64. Благодаря новой системе кэширования удалось добиться стабильной работы прокси с проигрывателями на основе VLC движка.
  • Достигнута полная совместимость прокси с проигрывателем IP-TV Player от BorPas.
  • Добавлена возможность в утилиту ZIPTVDump.exe производить копирование потока тв канала только до определённого момента времени X.
  • Добавлена возможность полного перевода ca авторизации прокси на новый сервер new.watch-tv.zet.
  • Реализована возможность проксирования M3U списка, совместимого с IP-TV Player от BorPas.
  • В прокси появился свой конфигурационный файл.

1.1 альфа

  • Модуль HttpListener удалён, сервер полностью переписан с использованием асинхронных сокетов, поэтому теперь может запускаться под обычным пользователем и права Администратора ему не нужны.
  • Поддерживает до 3-ёх одновременных сетевых подключений, то есть можно одновременно смотреть до трёх тв каналов.
  • Возможность простого запуска, без указания аргументов командной строки.
  • Увеличена производительность.

1.0 пред альфа

  • Разработана библиотека LibVLCPlugin.dll для взаимодействия с модулями VLC и плагином дескрамблинга libaccess_ca_plugin.dll из среды .NET.
  • В качестве серверного движка http используется модуль HttpListener.
  • Одновременно возможно только одно сетевое tcp подключение и, соответственно, передача одного потока канала.
  • Простая синхронная передача данных тв канала по tcp соединению.
  • Разработана утилита ZIPTVDump.exe для копирования потока тв канала в ts файл определённого размера.
